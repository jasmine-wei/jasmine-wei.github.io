版本控制工具（VCS）

Version Control System

版本控制工具，会将对于文件的每一次修改，都会当做是一个新的版本。
在版本控制工具后中，有一个叫做版本库的东西，里面存放着，所有的版本号和当前版本对应的文件内容。

版本控制工具可以非常方便的帮我们记录每一次的修改操作，修改的状态，也会保存下来，方便以后随时对于修改内容的恢复！！

版本控制工具的分类

  1. 本地式版本控制系统
    版本库存放在本地，只有本地才会有所有版本的历史记录，这中版本控制系统，只适合于单人开发。

  2. 集中式版本控制系统 （代表：SVN）
    版本库，集中存放在了源代码服务器上，所有人修改的历史，都会提交到服务器上生成新的版本，所有人在获取历史的时候，也都是从服务器获取

  3. 分布式版本控制系统 （代表：Git）
    版本库，分别存储在不同的程序员的不同的项目中，一个项目中可能包含很多个版本库，如果程序员之间想要共享版本库，需要通过版本库共享服务器！！

版本控制工具，主要的功能为： 记录修改历史，共享代码！！！

git的工作原理

三个区

  1. 工作目录
    指的就是项目文件存放的位置
  2. 暂存区
    通过git add就会将文件添加到这个暂存区
  3. 版本库
    存放版本历史信息的位置

git add 是将工作目录中的文件内容添加到了暂存区
git commit 是将暂存区中的内容在版本库中生成一条新的版本记录
git checkout 是将暂存区中的内容获取出来放到了工作目录
git reset –hard 是将版本库中指定版本的内容获取出来放到工作目录

一句话总结： git 其实就是将文件在各个区之间进行流转！！

文件的状态

  1. 新添加一个文件之后，这个文件属于 未追踪 的状态
  2. 将已经追踪的文件修改之后，这个文件属于 已修改 的状态
  3. 将已经追踪的文件删除之后，这个文件属于 已删除 的状态
  4. 将所有未追踪 已修改 已删除的文件进行一次 git add 操作之后，文件的状态会变成 待提交 状态
  5. 当执行一次git commit 操作之后，所有的文件都会是已提交的状态

冲突问题

分支A中创建了文件xx.txt, 分支B中也创建了个xx.txt，那么git就会发现，有两个同名的文件，这时候，就是冲突产生了，冲突产生之后，合并会失败，需要先解决冲突，然后再次提交，才能正常的合并。

解决冲突

冲突产生之后，git会告知哪个文件冲突了，然后在文件中，会用 >>>> ==== <<< 这样的方式将冲突的内容标记出来，我们只需要将标记删除，然后将代码改成自己想要的结果，再次add commit即可！！

分支的使用原则

只要有新的任务，就开辟一个新的分支去做，做完之后，将分支合并回去就可以了！！

  2. 在github的设置中 添加一个ssh key

  3. key中的内容就是将生成的公钥内容粘贴进去就可以了

  4. 以后使用github的时候，直接（只有）使用ssh连接就可以实现免密

github静态页面

  1. 在github中创建一个仓库
  2. 仓库的名字必须是 用户名.github.io
  3. 将写好的静态页面push到这个cnagku
  4. 通过 用户名.github.io 这个地址就可以访问网站了
